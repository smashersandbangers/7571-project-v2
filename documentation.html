Event Calendar Docs

application-settings.js

Purpose: Application wide settings configuration.

Global Variables

  • debug - turn on debug mode (verbose logging) for the application
  • preLoadEvent - turn on demo mode for application
  • preLoadedEvents - the JSON formatted string set for demo events

database.js

Purpose: Facilitate and abstract the transfer of information within the application.

Functions

  • addAttendeeToEvent(calendarName, eventid) - add a new attendee to a e-Brite event
  • addEventToCalendar(calendarName) - method to save a new event, event details are grabbed from input elements
  • getCalendarEvents(calendarName) - returns a JSON string of all events associated with local storage key value calendarName
  • getCalendarEventTitles(calendarName) - returns a simple array of event titles
  • getEventAttendees(calendarName, eventid) - returns the attendees array object associated with the event
  • getEvent(calendarName, eventid) - updated function signature - returns a single event, based on id, from the chosen calendar
  • getEventAttendees(calendarName, eventid) - returns a JSON string of event attendees
  • getNewEventid(calendarName) - returns the next available eventid for the specified calendar
  • saveCalendarEvents(calendarName, eventsObject) - places the events in local storage using key value calendarName
  • saveFullCalendarEvents(calendarName, fullCalendarEvents) - because FullCalendar events are far more complex than e-Brite events, we have created this function to take only what we want from a FullCalendar event.

Deprecated functions

  • getEvents() - use getCalendarEvents(calendarName) instead
  • getEvent(eventid) - returns a single event, in dev mode returns a defaultTestEvent

presentation.js

Purpose: Specific functions to display information on pages.

Functions

  • printAttendeesHTML(calendarName, eventid) - using the pass id of the event and associated calendar name, will attach a ul list to a id selector attendeesList
  • printEventDetailsHTML(calendarName, eventid) - prints the event details of a single events, uses id selector called eventDetails
  • printEventsHTML(calendarName) - printing the list of all Calendar Events, div with id of events needs to exist on page
  • printEventResults(calendarName) - displays a list of found events based on a match within the title, uses id selector eventResults

utility.js

Purpose: Everything else that doesn't have a home, generally helper functions.

Functions

  • addCalendarsToNavBar() - adds all existing calendar names in local storage not fully implemented yet
  • changeSelectedCalendar() - this gets called with the calendar drop down lists changes. It updates the FullCalendar with the new set of events.
  • getCurrentCalendar() - returns the currently selected calendarName value from the drop down list
  • getParamValue(paramName) - returns the matching value for the passed parameter name (paramName) from the URL. For example,
    • the following URL page.html?calendarName=demoEvents&eventid=02
    • getParamValue(calendarName) returns "demoEvents"
    • getParamVaule(eventid) returns "02"
  • loadApplication() - used to load and configure the FullCalendar on the landing page
  • log() - shorthand function for console.log()
